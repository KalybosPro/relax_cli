# Security Policy

Thank you for caring about the security of `relax_cli`.

## Reporting a Vulnerability

If you discover a security issue, please report it privately so we can address it before public disclosure.

Preferred report channels:

- Open a private issue on GitHub and mark it as a security report.
- Use the repository issue tracker and clearly label the report as security-related:
  - https://github.com/KalybosPro/relax_cli/issues

## What to Include

When reporting a vulnerability, include:

- A description of the issue.
- The version of `relax_cli` and any relevant environment details.
- Steps to reproduce the vulnerability.
- Any proof-of-concept code or minimal reproduction.

## Response

The maintainers will acknowledge your report as soon as possible and coordinate any fix or disclosure timeline.

## Supported Versions

Security fixes are typically applied to the current release and maintained versions of the project. If you are unsure whether your version is supported, please ask in your report.

## Disclosure

Please do not publicly disclose security issues until a fix has been released or the maintainers have agreed on a disclosure policy.
